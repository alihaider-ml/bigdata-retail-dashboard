In-Store Shopping Experience

Uncovering customer behavior with Big Data Analytics

Explore the Project

Phase 1: Problem Identification

Business Problem

  • Data from apps, stores, and POS is siloed.
  • Retailers can't answer questions about customer paths or high-traffic zones.
  • Decisions remain reactive, missing opportunities.

Proposed Solution

  • Integrate mobile app, geolocation, and transaction data.
  • Build a unified behavioral model.
  • Optimize store layout and product placement.

Data Sources

Mobile App Data

12M+ sessions, 220M+ events. Tracks clicks, views, cart events.

View Dataset

Transaction Data

1M+ transactions. Invoice, product, quantity, price, country.

View Dataset

Geolocation Data

92 days of indoor trajectories. Enables heatmaps, dwell-time analysis.

View Dataset

Phase 2: Pipeline Design

Step 1: Data Collection

Batch ingest mobile app, transaction, and geolocation datasets.

Step 2: Data Cleaning

Remove invalid records, normalize timestamps, smooth trajectories, encode fields.

Step 3: Data Integration

Align users across datasets temporally and spatially to create unified customer journey data.

Conclusion

  • Optimized store layouts
  • Data-driven product placement
  • Targeted promotions & personalized engagement
  • Predictive insights for purchase likelihood & hot zones
